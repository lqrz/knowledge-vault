# Q-Learning

Q-learning is an off-policy temporal-difference control algorithm for estimating optimal action values.

Its one-step tabular update is:

```text
Q(S_t, A_t) <- Q(S_t, A_t)
  + alpha * (R_{t+1} + gamma * max_a Q(S_{t+1}, a) - Q(S_t, A_t))
```

The target uses the greedy value of the next state, regardless of which action the behavior policy actually takes next.

## Off-Policy Control

Q-learning directly approximates the optimal action-value function `q_*`, while the behavior policy only determines which state-action pairs are visited and updated.

This is why Q-learning is off-policy: the learned target policy is greedy with respect to `Q`, but the data may be generated by a more exploratory policy such as epsilon-greedy behavior.

For correct tabular convergence, every state-action pair must continue to be updated, together with suitable step-size conditions.

## Contrast With Sarsa

[[temporal-difference-methods/sarsa|sarsa]] updates toward the value of the next action actually selected by the behavior policy. Q-learning updates toward the maximum action value in the next state.

When action selection is greedy, the target used by Sarsa and Q-learning can coincide. With exploratory behavior, they differ:

- Sarsa learns action values for the exploratory policy.
- Q-learning learns action values for the greedy target policy.

This matters for online performance in risky environments such as cliff walking.

## Maximization Bias

The `max` in Q-learning can introduce [[temporal-difference-methods/maximization-bias|maximization-bias]] when action-value estimates are noisy. [[temporal-difference-methods/double-q-learning|double-q-learning]] is one response.
