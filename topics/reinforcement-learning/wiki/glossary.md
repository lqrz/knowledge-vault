# Reinforcement Learning Glossary

This glossary is a staging area for short definitions. Promote a term into a focused concept page when it becomes central enough to reference from multiple notes.

## Terms

**Agent**: The decision-making system that selects actions.

**Environment**: The external process the agent interacts with through observations, actions, and rewards.

**State**: A representation of the situation used to describe what is relevant for future dynamics and rewards.

**Observation**: Information the agent receives from the environment; it may be a full state or a partial view of the state.

**Action**: A choice made by the agent that can influence the environment.

**Reward**: A scalar feedback signal used to define what behavior should be reinforced; see [[reward-vs-return]].

**Return**: The cumulative reward objective assigned to a trajectory, often discounted over time; see [[reward-vs-return]].

**Discount factor**: A parameter that controls how future rewards are weighted relative to immediate rewards.

**Episode**: One rollout or interaction sequence from a starting condition to termination or truncation.

**Trajectory**: A sequence of states or observations, actions, rewards, and sometimes policy probabilities collected during interaction.

**Policy**: A rule, function, or distribution that maps states or observations to actions.

**Value function**: A function that estimates expected return from a state under a policy.

**Action-value function**: A function that estimates expected return from taking an action in a state and then following a policy.

**Advantage**: A relative value signal that compares an action's return estimate against a baseline, often the state's value estimate.

**Markov decision process**: A formal sequential decision-making model with states, actions, transition dynamics, rewards, and a discount or horizon.

**Bellman equation**: A recursive relationship that connects the value of a state or action to immediate reward plus the value of future states.

**Prediction**: Estimating the value of a fixed policy; see [[prediction-and-control]].

**Control**: Finding or improving a policy to increase expected return; see [[prediction-and-control]].

**Exploration**: Trying actions to gather information about their consequences.

**Exploitation**: Selecting actions believed to produce high return based on current knowledge.

**Temporal-difference learning**: Learning from targets that combine observed rewards with bootstrapped estimates of future value; see [[temporal-difference-methods/overview|temporal-difference-learning]].

**TD prediction**: Policy evaluation with temporal-difference updates, especially one-step TD(0); see [[temporal-difference-methods/prediction|td-prediction]].

**TD error**: The gap between a current value estimate and a bootstrapped TD target; see [[temporal-difference-methods/td-error|td-error]].

**TD(0)**: The one-step TD prediction method that updates `V(S_t)` toward `R_{t+1} + gamma V(S_{t+1})`; see [[temporal-difference-methods/prediction|td-prediction]].

**Policy gradient**: A method family that optimizes policy parameters by estimating gradients of expected return.

**Monte Carlo method**: A sample-based method that estimates values by averaging returns from complete episodes; see [[monte-carlo-methods/overview|monte-carlo-methods]].

**Monte Carlo prediction**: Policy evaluation by averaging returns after visits to states or state-action pairs; see [[monte-carlo-methods/prediction|monte-carlo-prediction]].

**Monte Carlo control**: Policy improvement using action-value estimates learned from sampled returns; see [[monte-carlo-methods/control|monte-carlo-control]].

**First-visit Monte Carlo**: A Monte Carlo estimator that averages returns after the first visit to a state or state-action pair within each episode.

**Every-visit Monte Carlo**: A Monte Carlo estimator that averages returns after every visit to a state or state-action pair.

**Exploring starts**: An assumption that every state-action pair has nonzero probability of being selected as an episode start; see [[monte-carlo-methods/exploring-starts|exploring-starts]].

**On-policy method**: A method that evaluates or improves the same policy used to generate behavior.

**Off-policy method**: A method that learns about a target policy from data generated by a different behavior policy; see [[monte-carlo-methods/off-policy-control|off-policy-monte-carlo]].

**Target policy**: The policy being evaluated or improved in off-policy learning.

**Behavior policy**: The policy used to generate data in off-policy learning.

**Coverage**: The requirement that the behavior policy assigns nonzero probability to actions that the target policy may take.

**Importance sampling**: A correction technique for estimating expectations under one distribution using samples from another; see [[monte-carlo-methods/importance-sampling|importance-sampling]].

**Ordinary importance sampling**: An off-policy estimator that averages returns multiplied by importance-sampling ratios.

**Weighted importance sampling**: An off-policy estimator that normalizes ratio-weighted returns by the accumulated importance weights.

**Sarsa**: An on-policy TD control algorithm that updates an action value using the next action actually selected by the behavior policy; see [[temporal-difference-methods/sarsa|sarsa]].

**Q-learning**: An off-policy TD control algorithm that updates action values toward the greedy next-action value; see [[temporal-difference-methods/q-learning|q-learning]].

**Expected Sarsa**: A TD control algorithm that updates toward the expected next action value under a policy; see [[temporal-difference-methods/expected-sarsa|expected-sarsa]].

**Maximization bias**: Positive bias caused by using the maximum of noisy value estimates as an estimate of the maximum true value; see [[temporal-difference-methods/maximization-bias|maximization-bias]].

**Double Q-learning**: A Q-learning variant that uses two action-value estimates to reduce maximization bias; see [[temporal-difference-methods/double-q-learning|double-q-learning]].

**Afterstate**: A post-action situation after the known immediate effect of an action but before the remaining environment response; see [[temporal-difference-methods/afterstates|afterstates]].
