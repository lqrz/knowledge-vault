# Machine Learning Glossary

This glossary is a staging area for short definitions. Promote a term into a focused concept page when it becomes central enough to reference from multiple notes.

## Terms

**Dataset**: A collection of examples used for training, validation, testing, analysis, or deployment monitoring.

**Example**: One observed case from a dataset, often represented as input features and, for supervised learning, an associated label or target.

**Feature**: A measurable or derived input variable provided to a model.

**Label**: The observed target value used to supervise a prediction task.

**Model**: A parameterized or non-parameterized function, rule, or system used to make predictions, assign scores, generate outputs, or estimate structure from data.

**Parameter**: A learned value inside a model that changes during training.

**Loss function**: A training objective that assigns a numerical penalty to model outputs.

**Empirical risk minimisation**: The principle of choosing a model that minimises average loss on a finite training dataset; see [[empirical-risk-minimisation]].

**Optimization**: The process of adjusting model parameters or choices to improve an objective.

**Generalization**: A model's ability to perform well on data beyond the specific examples used to fit it.

**Overfitting**: A failure mode where a model fits training-specific noise or quirks and performs worse on new data.

**Underfitting**: A failure mode where a model is too limited, poorly trained, or poorly specified to capture important structure in the data.

**Evaluation metric**: A measurement used to judge model behavior or task performance.
